## 生产环境部署前必做事项

### 1. 环境变量配置 ✅

- [ ] 生成强JWT密钥: `openssl rand -base64 64 | tr -d '\n'`
- [ ] 更新 `.env` 文件中的 `JWT_SECRET`
- [ ] 设置 `NODE_ENV=production`
- [ ] 配置正确的 `FRONTEND_URL`
- [ ] 确保没有提交 `.env` 文件到版本控制

### 2. 服务器安全配置 ✅

- [ ] 启用防火墙 (UFW/iptables)
- [ ] 配置HTTPS (Let's Encrypt)
- [ ] 设置非root用户运行服务
- [ ] 配置日志轮转
- [ ] 设置文件权限: `chmod 600 db/linkhub.db`

- [ ] 运行安全检查: `npm run security-check`

- [ ] 移除开发调试代码
- [ ] 验证输入验证是否生效

### 4. 数据库安全 ✅

- [ ] 备份现有数据库

### 5. 监控和日志 ✅

- [ ] 设置错误监控 (Sentry等)

## 安全配置详情

### JWT密钥要求

- 长度至少32字符
- 使用强随机生成器
- 定期轮换密钥 (建议每90天)
- 不同环境使用不同密钥

### 输入验证规则

1. **邮箱**: 必须符合标准邮箱格式
2. **密码**: 至少8字符，包含大小写字母和数字
3. **URL**: 必须有效URL格式，仅允许http/https协议
4. **用户名**: 3-20字符，仅允许字母、数字、下划线
5. **XSS防护**: 所有用户输入都经过HTML实体编码

### 生产环境禁用功能

- ❌ 开发模式token显示 (`DEV_SHOW_TOKENS`)
- ❌ 敏感数据日志记录
- ❌ 调试信息输出
- ❌ 默认密码和密钥

### 推荐的安全中间件

```bash
npm install helmet express-rate-limit
```

在 `index.js` 中添加:

```javascript
const helmet = require('helmet');
const rateLimit = require('express-rate-limit');

app.use(helmet());

const limiter = rateLimit({
  windowMs: 15 * 60 * 1000, // 15分钟
  max: 100, // 每个IP限制100个请求
  message: '请求过于频繁，请稍后再试',
});
app.use('/api/', limiter);
```

## 应急响应计划

### 发现安全漏洞时

1. **立即行动**: 暂停受影响服务
2. **调查**: 确定漏洞范围和影响
3. **修复**: 应用安全补丁
4. **通知**: 告知受影响用户
5. **复盘**: 分析根本原因，防止再次发生

### 数据泄露响应

1. **确认泄露**: 验证泄露的数据类型和范围
2. **法律合规**: 根据当地法规通知监管机构
3. **用户通知**: 透明地告知受影响用户
4. **补救措施**: 提供身份保护服务
5. **系统加固**: 加强安全措施
