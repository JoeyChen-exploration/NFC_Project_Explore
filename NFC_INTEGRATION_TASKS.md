# NFC功能集成到EditorPage任务文档

## 📊 问题分析

### 当前设计不一致

1. **NFC标签**: 点击跳转到 `/nfc` 独立页面
2. **其他标签**: 点击切换EditorPage内的内容
3. **用户体验**: 导航行为不一致，显得设计粗糙
4. **界面美观**: 破坏统一的单页应用体验

### 设计目标

1. **统一导航**: 所有标签行为一致
2. **流畅体验**: 保持单页应用的编辑流程
3. **功能完整**: NFC管理功能保持不变
4. **界面美观**: 统一的标签切换交互

## 🎯 解决方案

### 方案: 将NFC功能集成到EditorPage

- 创建 `NfcEditor` 组件，作为EditorPage的一个标签内容
- 移除NFC页面跳转逻辑
- 保持现有的NFC管理功能完整
- 统一所有标签的切换行为

### 技术实现

1. **创建NfcEditor组件**: 将NfcPage功能迁移到组件
2. **更新EditorPage导航**: 添加NFC标签，移除跳转
3. **集成到EditorPage**: 作为标签内容渲染
4. **移除独立NfcPage**: 保持路由但重定向或移除

## 📁 实施任务

### 任务1: 创建NfcEditor组件

**文件**: `src/components/editor/NfcEditor.jsx`

**功能**: NFC卡片管理组件，集成到EditorPage

- 卡片列表展示
- 绑定新卡片表单
- 卡片操作（重命名、激活/停用、解绑）
- 扫描统计展示

**接口**:

```jsx
<NfcEditor
  user={user} // 用户信息
  onCardBind={data => {}} // 绑定卡片回调
  onCardUpdate={(id, data) => {}} // 更新卡片回调
  onCardDelete={id => {}} // 删除卡片回调
/>
```

**设计要求**:

- 响应式设计，适配EditorPage布局
- 复用现有的UI组件（Card、FormField等）
- 保持NfcPage的所有功能
- 移动端适配良好

### 任务2: 更新EditorPage导航配置

**文件**: `src/pages/EditorPage.jsx`

**修改内容**:

1. 更新NAV_TABS配置，添加NFC标签
2. 移除NFC跳转逻辑
3. 添加NFC标签的渲染内容

**当前NAV_TABS**:

```jsx
const NAV_TABS = [
  { id: 'links', label: 'Links' },
  { id: 'appearance', label: 'Appearance' },
  { id: 'nfc', label: 'NFC' }, // 当前会跳转
```

**目标NAV_TABS**:

```jsx
const NAV_TABS = [

```

### 任务3: 更新EditorPage渲染逻辑

**文件**: `src/pages/EditorPage.jsx`

**修改内容**:

1. 导入NfcEditor组件
2. 在标签切换逻辑中添加NFC case
3. 渲染NfcEditor作为标签内容

**当前渲染逻辑**:

```jsx
// 需要修改switch语句，添加NFC case
switch (tab) {
  case 'links':
    return <LinkList ... />;
  case 'appearance':
    return ...; // 主题和社交
  case 'nfc':
    // 当前: 跳转到/nfc页面
    // 目标: 渲染NfcEditor组件
  case 'settings':
    return <SettingsPanel ... />;
}
```

### 任务4: 更新EditorNav组件

**文件**: `src/components/editor/EditorNav.jsx`

**修改内容**:

1. 移除NFC跳转的特殊处理
2. 统一所有标签的点击行为
3. 保持现有的样式和交互

**当前逻辑**:

```jsx
const handleTabClick = tabId => {
  if (tabId === 'nfc') {
    // 特殊处理: 跳转到NFC页面
    navigate('/nfc');
  } else {
    // 普通标签切换
    onTabChange(tabId);
  }
};
```

**目标逻辑**:

```jsx
const handleTabClick = tabId => {
  // 所有标签统一行为: 切换内容
  onTabChange(tabId);
};
```

### 任务5: 处理独立NfcPage路由（可选）

**选项A: 保留并重定向**

- 保留 `/nfc` 路由

- 保持现有书签和链接可用

**选项B: 移除独立页面**

- 删除NfcPage.jsx文件
- 移除 `/nfc` 路由配置
- 简化项目结构

**推荐选项A**: 保留兼容性，用户书签和外部链接仍然有效
